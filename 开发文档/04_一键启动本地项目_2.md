这是一个非常关键的步骤，涉及到如何**组织代码**。在 Go 语言中，函数可以写在同一个文件里，也可以分开。

为了让你操作最简单，我建议你直接将这个函数写在 `main.go` 中。

### 1. 放置位置
请看下面的结构，你需要把 `setupRoutes` 函数放在 `main` 函数的**外面**（通常放在 `main` 函数下方），然后在 `main` 函数里**调用**它。

你的 `main.go` 应该长这样：

```go
package main

import (
	"github.com/gin-gonic/gin"
	"net/http"
	"os"      // 新增
	"os/exec"
	"runtime"
)

func main() {
	// 1. 检查命令行参数 (bobo dev)
	if len(os.Args) < 2 || os.Args[1] != "dev" {
		println("请使用 'bobo dev' 来启动服务")
		return
	}

	InitDB()
	defer DB.Close()

	r := gin.Default()
	r.LoadHTMLGlob("templates/*")

	// 2. 调用我们定义的路由配置函数
	setupRoutes(r)

	r.Run(":5051")
}

// --- 把函数放在这里 ---
func setupRoutes(r *gin.Engine) {
	// 首页：展示项目
	r.GET("/", func(c *gin.Context) {
		rows, _ := DB.Query("SELECT id, name, path, command, note FROM projects")
		var projects []Project
		for rows.Next() {
			var p Project
			rows.Scan(&p.ID, &p.Name, &p.Path, &p.Command, &p.Note)
			projects = append(projects, p)
		}
		c.HTML(http.StatusOK, "index.html", gin.H{
			"Projects": projects,
		})
	})

	// 添加项目接口
	r.POST("/add", func(c *gin.Context) {
		name := c.PostForm("name")
		path := c.PostForm("path")
		cmd := c.PostForm("cmd")
		note := c.PostForm("note")
		DB.Exec("INSERT INTO projects (name, path, command, note) VALUES (?, ?, ?, ?)", name, path, cmd, note)
		c.Redirect(http.StatusMovedPermanently, "/")
	})

	// 核心启动接口
	r.GET("/run/:id", func(c *gin.Context) {
		id := c.Param("id")
		var p Project
		err := DB.QueryRow("SELECT path, command FROM projects WHERE id = ?", id).Scan(&p.Path, &p.Command)
		if err != nil {
			c.String(http.StatusNotFound, "找不到该项目")
			return
		}

		var cmd *exec.Cmd
		if runtime.GOOS == "windows" {
			cmd = exec.Command("cmd", "/C", p.Command)
		} else {
			cmd = exec.Command("sh", "-c", p.Command)
		}
		cmd.Dir = p.Path

		err = cmd.Start() // 异步启动
		if err != nil {
			c.String(http.StatusInternalServerError, "启动失败: "+err.Error())
			return
		}
		c.Redirect(http.StatusMovedPermanently, "/")
	})
}
```

---

### 2. 为什么要这样写？（原理说明）



* **解耦（Decoupling）：** 随着功能变多（比如以后要加“删除项目”、“修改项目”），如果全部塞在 `main` 函数里，代码会变得非常长，难以阅读。把路由（URL 和功能的对应关系）单独拎出来，代码更整洁。
* **作用域：** 在 `main` 函数里我们创建了变量 `r`（Gin 引擎）。通过 `setupRoutes(r)`，我们将这个引擎的控制权“借给”了 setupRoutes 函数，让它去配置具体的规则。

### 3. 编译并使用 `bobo dev`

既然你现在加了 `os.Args` 的判断，运行方式会发生一点变化：

1.  **临时运行：** 你不能只输入 `go run .` 了，必须带上参数：
    ```bash
    go run . dev
    ```
2.  **永久使用：** 编译它：`go build -o bobo.exe`
    然后你就可以在终端直接输入：`./bobo dev` 启动了！

**试试看！如果运行 `go run . dev` 能够正常打开并点击“启动”按钮运行你的本地脚本，那么你的 Bobo 工具就正式完工了！**
